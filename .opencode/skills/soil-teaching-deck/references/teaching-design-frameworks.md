# 教學設計框架工具箱

本文件收錄四套進階教學設計框架，可與 SOIL 六引擎搭配使用。
每套框架解決不同層面的問題——選擇適合的框架疊加即可，不需要全部使用。

---

## 框架總覽：誰解決什麼問題？

| 框架 | 核心問題 | 一句話 | 與 SOIL 的關係 |
|------|---------|--------|---------------|
| **UbD** | 「教什麼才對？」 | 以終為始，先定結果再設計內容 | 影響 Engine 1（概念定位）的方向 |
| **Merrill** | 「怎麼教才會用？」 | 圍繞真實任務，從看到做到 | 影響 Engine 2-3（情境+頁面架構） |
| **Gagné** | 「課堂節奏怎麼控？」 | 九大事件確保注意力不掉線 | 精細化 Engine 2（情境定位）的每一步 |
| **ARCS** | 「學生沒動力怎麼辦？」 | 用注意、相關、信心、滿足調味 | 為所有引擎加上「動機潤滑劑」 |

### 何時疊加哪個框架？

```
你的情境是...                              → 疊加這個框架
────────────────────────────────────────────────────────
不確定該教什麼、怕教錯重點                   → UbD
教實務技能（寫程式、操作工具、解決問題）       → Merrill
聽眾注意力容易跑掉（長課程、下午場）          → Gagné
內容枯燥但不得不教（法規、SOP、合規）         → ARCS
一般教學、已經知道教什麼                     → 純 SOIL 即可
```

## 1. UbD 重理解的課程設計（Backward Design）

### 核心理念：以終為始

不是從「我有什麼教材」出發，而是從「學生最後該帶走什麼」倒推。

### 三個階段

```
階段一                    階段二                    階段三
確認預期結果        →     決定評量證據        →     安排學習活動
「學生帶走什麼？」        「如何證明他懂了？」       「為了通過評量，要教什麼？」
```

#### 階段一：確認預期結果

```markdown
## UbD 階段一：預期結果

### 持久理解（Enduring Understanding）
[課程結束 3 個月後，學生仍然記得的核心理解是什麼？]

### 核心問題（Essential Questions）
1. [引導學生思考的開放式問題]
2. [...]

### 學生將知道（Knowledge）
- [具體事實或概念]

### 學生將能做到（Skills）
- [可觀察、可衡量的能力]
```

#### 階段二：決定評量證據

```markdown
## UbD 階段二：評量證據

### 表現任務（Performance Task）
- 任務描述：[例：現場寫出一個正確的 GAS 腳本]
- 情境：[在什麼條件下完成]
- 評量標準：[如何判定「做到了」]

### 其他證據
- 隨堂小測驗
- 課堂討論表現
- 反思日誌
```

#### 階段三：安排學習活動

到這一步才開始設計教學內容——交給 SOIL 六引擎執行。

### UbD → SOIL 的銜接

| UbD 產出 | 輸入到 SOIL 哪個引擎 |
|---------|-------------------|
| 持久理解 | Engine 1「帶走一句話」 |
| 核心問題 | Engine 2「引起動機」的破題素材 |
| 表現任務 | Engine 3 的實作頁/練習頁設計 |
| 評量標準 | Engine 6 驗證時的自查清單 |

### 適用情境
- ✅ 第一次設計這門課（不確定該教什麼）
- ✅ 學科導向課程（數學、科學、語言）
- ✅ 課程改版（重新思考「到底要教什麼」）
- ❌ 不適合已有明確教材只需排版的場景

## 2. Merrill 梅里爾：教學首要原則

### 核心理念：問題中心

整堂課圍繞一個「學員馬上會在工作上遇到的真實任務」展開。

### 五個階段

```
問題中心 → 激活舊知 → 展示範例 → 嘗試應用 → 整合遷移
 Problem    Activation  Demonstration  Application  Integration
```

#### 各階段對應的投影片設計

| Merrill 階段 | 投影片設計建議 | SOIL 引擎 |
|-------------|-------------|----------|
| **問題中心** | 開場展示真實工作場景/痛點。用全版圖片或短影片 | Engine 2（引起動機） |
| **激活舊知** | 「你目前怎麼處理這件事？」引導討論。用對比表格 | Engine 2（引起動機） |
| **展示範例** | 老師 live demo 或錄影截圖。用左圖右文分割排版 | Engine 3（頁面架構）+ Engine 4（步驟化） |
| **嘗試應用** | 給學員練習題。用互動頁或留白頁 | Engine 3（頁面架構） |
| **整合遷移** | 「回到你的工作，這招怎麼用？」反思頁 | Engine 2（喚起行動） |

#### Merrill 頁面架構範例（以 10 頁簡報為例）

```
Page 1:   封面
Page 2:   真實問題場景（問題中心）
Page 3:   「你目前怎麼做？」（激活舊知）
Page 4-5: 正確做法示範（展示範例）
Page 6:   步驟拆解（展示範例）
Page 7-8: 練習題/實作（嘗試應用）
Page 9:   「帶回你的工作」（整合遷移）
Page 10:  總結 + 資源連結
```

### 適用情境
- ✅ 職業技能培訓（寫程式、操作軟體、使用工具）
- ✅ 企業內訓（流程改善、新系統導入）
- ✅ 工作坊（hands-on 實作型課程）
- ❌ 不適合純知識型/概念講座

## 3. Gagné 蓋聶：教學九大事件

### 核心理念：微觀課堂控管

把一堂課拆成 9 個「注意力節點」，確保每個環節都精準觸發學習反應。

### 九大事件與投影片對應

| # | 事件 | 目的 | 投影片設計建議 | SOIL 位置 |
|---|------|------|-------------|----------|
| 1 | **引起注意** | 抓住目光 | 震撼圖片、出人意料的數據、提問 | 引起動機 Page 1-2 |
| 2 | **告知目標** | 設定期待 | 「今天學完你將能...」清單 | 引起動機 Page 2 |
| 3 | **喚起舊知** | 連結已知 | 「你知道 XXX 嗎？」互動提問 | 引起動機 Page 2-3 |
| 4 | **呈現教材** | 傳遞新知 | 核心內容頁，一頁一觀點 | 維持注意 Page 3-6 |
| 5 | **提供學習引導** | 輔助理解 | 圖解、比喻、案例、step-by-step | 維持注意 Page 4-7 |
| 6 | **引發表現** | 讓學員動手 | 練習題、實作挑戰、小組討論 | 維持注意 Page 7-8 |
| 7 | **提供回饋** | 確認正確性 | 解答頁、常見錯誤對比 | 維持注意 Page 8 |
| 8 | **評量表現** | 檢驗成效 | 隨堂測驗、現場實作 | 喚起行動 Page 9 |
| 9 | **加強記憶與遷移** | 持久記住 | 總結、口訣、實務應用場景 | 喚起行動 Page 9-10 |

### Gagné → SOIL Engine 2 的精細化

SOIL 的三段式節奏（引起動機 → 維持注意 → 喚起行動）可以用 Gagné 九事件來「展開」：

```
SOIL 引起動機（10-15%）
  └── Gagné #1 引起注意
  └── Gagné #2 告知目標
  └── Gagné #3 喚起舊知

SOIL 維持注意（65-75%）
  └── Gagné #4 呈現教材
  └── Gagné #5 提供學習引導
  └── Gagné #6 引發表現
  └── Gagné #7 提供回饋

SOIL 喚起行動（15-20%）
  └── Gagné #8 評量表現
  └── Gagné #9 加強記憶與遷移
```

### 適用情境
- ✅ 長時間課程（需要精準控管節奏）
- ✅ 線上教學（注意力更容易分散）
- ✅ 新手講師（需要明確的流程指引）
- ❌ 不適合即興演講或腦力激盪工作坊

## 4. ARCS 學習動機模型

### 核心理念：給枯燥內容「調味」

當課程內容本身不夠吸引人時，用四個維度來增加學習動機。

### 四個維度

| 維度 | 英文 | 核心問題 | 投影片設計策略 |
|------|------|---------|-------------|
| **A** 注意 | Attention | 「為什麼要看這個？」 | 視覺震撼、出人意料的數據、奇聞軼事、反直覺事實 |
| **R** 相關 | Relevance | 「這跟我有什麼關係？」 | 連結學員的工作/考績/獎金/日常生活 |
| **C** 信心 | Confidence | 「我學得會嗎？」 | 拆解困難任務、提供鷹架、展示「跟你一樣的人也學會了」 |
| **S** 滿足 | Satisfaction | 「學了有什麼好處？」 | 即時獎勵、成就解鎖、實際應用成果展示 |

### ARCS 在每頁投影片的應用

```markdown
## ARCS 自檢清單（每頁投影片）

- [ ] A：這頁有沒有「讓人想繼續看」的元素？
- [ ] R：學員能看出「這跟我的工作/生活有什麼關係」嗎？
- [ ] C：如果內容很難，有沒有降低門檻的設計（分步驟、給提示）？
- [ ] S：學到這裡，學員有沒有「我好像學到了什麼」的感覺？
```

### ARCS 在 SOIL 各引擎的滲透

| SOIL 引擎 | ARCS 主要滲透維度 | 範例 |
|-----------|-----------------|------|
| Engine 1 概念定位 | R（相關） | 選擇與學員切身相關的核心概念 |
| Engine 2 引起動機 | A（注意）| 用震撼開場、反直覺數據 |
| Engine 2 維持注意 | C（信心）| 適當拆解、中途小勝 |
| Engine 2 喚起行動 | S（滿足）| 展示學習成果、給予肯定 |
| Engine 3 頁面架構 | A + C | 每頁都有視覺焦點 + 難度適中 |
| Engine 4 認知編輯 | C（信心）| 降雜訊 = 降低認知負荷 = 增加信心 |

### 適用情境
- ✅ 法規/合規/SOP 等枯燥必修內容
- ✅ 學員被「強迫」來上課的場景
- ✅ 線上非同步課程（沒有老師在場維持注意力）
- ❌ 學員本身就高度有動力的場景（可能感覺多餘）

## 框架組合建議

### 組合 A：完整課程設計（最全面）

```
ADDIE（生命週期管理）
  └── A 分析 → UbD 階段一（確認預期結果）
  └── D 設計 → UbD 階段二（決定評量證據）
  └── D 發展 → SOIL 六引擎 + Gagné 九事件（精細化頁面節奏）
  └── I 執行
  └── E 評估
```

### 組合 B：技能培訓課程

```
Merrill 五階段（決定內容結構）
  └── SOIL 六引擎（製作投影片）
  └── ARCS 自檢（確保動機不掉線）
```

### 組合 C：法規/SOP 課程

```
SOIL 六引擎（製作投影片）
  └── ARCS 調味（每頁檢查四維度）
  └── Gagné #1-3（強化開場吸引力）
```

### 組合 D：快速出投影片

```
SOIL 六引擎（直接製作）
不需要額外框架
```
